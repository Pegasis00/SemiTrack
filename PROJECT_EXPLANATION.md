# SemiTrack India: Project Deep Dive & Explanation

This document provides a comprehensive walkthrough of the **SemiTrack India** project. It explains what every file does, where the data comes from, and how the "Substitution Tracker" actually works.

---

## 1. Data Guide: What's in the folders?

The project uses three types of data stored in `data/`:

### 📂 `data/raw/`

* **`india_imports_8542_3818_filtered.csv`**: This is a direct extract from the **CEPII BACI HS92** global trade database. It contains 6,261 rows of transactions where India is the importer for two specific product categories:
  * **HS 8542:** Electronic Integrated Circuits (finished chips).
  * **HS 3818:** Doped Silicon Wafers (raw inputs for chip packaging).

### 📂 `data/processed/`

* **`india_semiconductor_integrated_annual.csv`**: A summary table (30 rows, 1995–2024) showing total yearly imports, China's market share, and "policy dummies" (binary 0 or 1 flags for years with COVID, PLI schemes, etc.).
* **`india_semiconductor_country_year_breakdown.csv`**: A detailed breakdown of which countries supplied chips to India every year. This is used to build the Supplier Risk matrix.
* **`india_semiconductor_integrated_annual_enriched.csv`**: Generated by Step 1. It adds math-heavy features like "log-differences" and "rolling averages" that the AI models need to make accurate forecasts.

### 📂 `data/synthetic/`

* **`actual_imports_2025_2026.csv`**: Since we don't have real 2025 data yet, this "synthetic" file simulates what would happen if the **Micron ATMP plant** successfully substitute imports. It is used to demonstrate the dashboard's power.

## 2. Pipeline Walkthrough: The 6 Steps

The `src/` folder contains the "engine" of the project. You run them using `python run_pipeline.py`.

### Step 1: `01_preprocessing.py`

Reads the raw data and "enriches" it. It calculates things like the **Log-Real Import value** (which smooths out inflation and weird spikes) so the forecasting models don't get confused by massive 90s inflation.

### Step 2: `02_eda.py`

Generates 6 Exploratory Data Analysis (EDA) charts. These show you the "big picture": how imports grew from **$237M in 1995** to over **$23B in 2024**.

### Step 3: `03_stationarity.py`

This is a "sanity check" script. It runs statistical tests (ADF/KPSS) to prove that the data is steady enough to forecast. It also runs a **Chow Test**, which proves that the year **2018** was a "structural break" where India's imports started growing much faster than before.

### Step 4: `04_arima.py`

Builds the **Baseline AI Model**. It tries hundreds of different settings to find the best "univariate" forecast (forecasting based only on past patterns).

### Step 5: `05_arimax.py`

Builds the **Advanced AI Model**. "ARIMAX" is like ARIMA, but it also considers **External Factors** (the "X"). It looks at the policy dummies (PLI schemes, Micron MoU) to see if government policy actually boosted imports. **This is our preferred "Business-As-Usual" (BAU) model.**

### Step 6: `06_mixshift.py`

This is the **Substitution Logic**. It takes the total BAU forecast and splits it into two expectations:

1. How many finished chips (`8542`) we expect to import.
2. How many raw wafers (`3818`) we expect to import.

---

## 3. The "Mix Shift" Secret: How we detect substitution

The **Micron ATMP plant** in Gujarat doesn't "make" chips from scratch; it **packages** them. This creates a very specific "signature" in trade data that this project is designed to find:

1. **HS 8542 Falls:** If we package chips in India, we stop importing as many finished ones.
2. **HS 3818 Rises:** To package them here, we must import more raw wafers/inputs.

**The Logic:**

* If *both* go down, it's just a general economic recession.
* If *both* go up, India is just buying more electronics.
* **But if 8542 goes DOWN and 3818 goes UP simultaneously**, that is the "Mix Shift" proof that domestic packaging is substituting imports.

---

## 4. Key Metrics Explained

* **HHI (Herfindahl-Hirschman Index):** A score from 0 to 1. 0 means you buy from many countries equally (safe). 1 means you buy from only one country (risky). India's HHI has been rising, showing we are becoming **more** dependent on China and Taiwan.
* **Real Value (2015 USD):** Adjusts current dollar amounts for inflation to show the "true" volume of chips imported.
* **Geo-Risk Score:** A custom score (0-10) we calculate by multiplying every country's market share by its geopolitical risk (e.g., China=9.0, USA=3.5).

---

## 5. How to use this project

1. **Train the models:** Run `python run_pipeline.py`. This generates all technical charts and forecast reports in the `outputs/` folder.
2. **Launch the Dashboard:** Run `python app.py`.
3. **Test the AI:** In the dashboard, go to the **Substitution Tracker** tab and upload `data/synthetic/actual_imports_2025_2026.csv`.
4. **Observe the Verdict:** The dashboard will compare the "actuals" in that file vs the AI's "BAU Forecast" and tell you if substitution is happening.
